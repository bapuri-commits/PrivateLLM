## 1. 프로젝트 목표와 철학

### 최종 목표

상용 AI(ChatGPT, Midjourney 등)와 유사한 수준의 AI를 **검열 없이, 개인 환경에서** 운용하는 것.

### 핵심 철학: "밑바닥이 아니라 조립"

| 접근 방식 | 설명 | 이 프로젝트의 선택 |
|----------|------|-----------------|
| **바닥부터 구축** | 데이터 수집 → 전처리 → 파인튜닝 → 평가 → 서빙 | X (나중에 별도 학습 프로젝트로) |
| **기존 제품 조합** | 이미 잘 만들어진 uncensored 모델/도구를 설치하고 바로 사용 | **O (이 프로젝트)** |

이유:
- 이미 커뮤니티에서 검열 제거된 고품질 모델이 다수 존재
- 먼저 "검열 없는 AI" 자체를 경험하고, 부족한 점을 파악한 뒤에 강화하는 게 효율적
- 파인튜닝/ML 기초는 별도 학습 프로젝트에서 체계적으로 배움 (부록 참조)

### 다루는 영역

| 영역 | 구체적 용도 | 우선순위 |
|------|-----------|---------|
| **텍스트 LLM** | 검열 없는 대화, 질의응답, 코딩 어시스턴트, 창작/RP | 1순위 (Phase 1) |
| **이미지 생성/편집** | 사진 생성, 편집, 인체 구조/포즈/표정 이해 | 2순위 (Phase 2) |
| **영상 처리** | 이미지→동영상 변환, 기존 영상 보정/효과 | 3순위 (Phase 3) |
| **AI 영상 생성** | AI로 영상을 처음부터 생성 | 4순위 (Phase 5+) |

## 2. 핵심 개념 정리

### LLM (Large Language Model)

대규모 텍스트로 학습된 언어 모델. ChatGPT, Claude의 핵심 엔진.
이 프로젝트에서는 오픈소스 LLM을 로컬 PC에서 직접 실행합니다.

### "Uncensored" / "Abliterated"

- **Censored 모델**: 상용 모델은 특정 주제에 "죄송합니다, 그건 답변할 수 없습니다" 식의 거절을 하도록 학습됨
- **Uncensored 모델**: 이 거절 메커니즘이 제거된 모델
- **Abliteration**: 모델 가중치에서 "거절 방향"을 수학적으로 찾아 제거하는 기법. 재학습 없이 변환 가능

### 양자화 (Quantization)

모델의 가중치를 더 적은 비트로 표현하여 메모리를 줄이는 기법.

- **FP16** (16비트): 원본. 8B 모델 ≈ 16GB 필요
- **Q4** (4비트): 75% 절감. 8B 모델 ≈ 4~5GB 필요. 품질 손실 미미
- **GGUF**: llama.cpp 및 Ollama가 사용하는 양자화 포맷

### VRAM (Video RAM)

GPU 전용 메모리. 모델 가중치 + 연산 중간 결과가 여기에 올라감.
VRAM이 부족하면 모델이 로딩되지 않거나 크래시(OOM)가 발생.

### Stable Diffusion

오픈소스 이미지 생성 AI. 텍스트 프롬프트를 입력하면 이미지를 생성.
상용 서비스(Midjourney, DALL-E)와 달리, 로컬에서 실행하면 **safety checker를 끌 수 있어서** 어떤 이미지든 생성 가능.

### LoRA (Low-Rank Adaptation)

모델 전체를 재학습하지 않고, 작은 "어댑터"만 학습시켜 특정 스타일/능력을 추가하는 기법.
이미지 쪽에서 특히 많이 쓰임 (예: 특정 화풍, 특정 캐릭터 얼굴 학습).

### ControlNet

이미지 생성 시 **포즈, 윤곽, 깊이** 등을 제어하는 확장 모델.
"이 포즈로 이 사람을 그려줘" 같은 세밀한 제어가 가능해짐.
인체 구조/구도/역동적 움직임 표현에 핵심적인 도구.

## 4. Phase 1 — 텍스트 LLM 즉시 구동

> 목표: 검열 없는 LLM과 대화할 수 있는 환경을 **30분 안에** 확보
> 비용: $0 (로컬)

### Ollama란?

오픈소스 LLM을 **한 줄 명령어로** 다운로드하고 실행할 수 있는 도구.
모델 다운로드, 양자화, 메모리 관리, CLI 대화 인터페이스를 전부 알아서 처리합니다.

- 공식 사이트: https://ollama.com
- Windows/Mac/Linux 모두 지원

### 설치 및 실행

```bash
# 1. Ollama 설치 (공식 사이트에서 다운로드 후 설치)

# 2. uncensored 모델 실행 (자동 다운로드)
ollama run dolphin-llama3

# 3. 바로 대화 시작
>>> 아무 질문이나 입력
```

### 추천 모델 (Ollama에서 바로 실행 가능)

| 모델 | 명령어 | 크기 | 특징 |
|------|--------|------|------|
| **Dolphin Llama 3** | `ollama run dolphin-llama3` | ~4.7GB | 순종성 높음, 거절 안 함 |
| **Llama 3.2 Uncensored** | `ollama run artifish/llama3.2-uncensored` | ~2.2GB | 가벼움, 128K 컨텍스트 |
| **Llama 3.3 Abliterated (70B)** | `ollama run huihui_ai/llama3.3-abliterated` | ~40GB | 최고 품질 (RunPod 필요) |

### Phase 1 체크리스트

- [ ] Ollama 설치
- [ ] Dolphin Llama 3 실행 및 대화 테스트
- [ ] 거절 없이 응답하는지 다양한 주제로 확인
- [ ] 한국어 응답 품질 평가
- [ ] 코딩 어시스턴트로서의 품질 평가
- [ ] 창작/RP 능력 평가
- [ ] **이 LLM에게 Phase 2~4 강화 방향 상담** ← 핵심!

### 이 Phase에서 파악할 것

- 8B uncensored 모델의 한계가 어디인지
- 한국어 품질이 충분한지, 부족하면 어떻게 부족한지
- 어떤 영역에서 강화가 필요한지 (→ Phase 4의 입력)

## 5. Phase 2 — 이미지 생성/편집 환경 구축

> 목표: 검열 없는 이미지 생성 + 인체 구조/포즈 제어 가능한 환경 확보
> 비용: $0 (로컬)

### 이미지 생성 AI의 구조 (초보자 가이드)

이미지 생성 AI는 텍스트 LLM과 완전히 다른 종류의 모델입니다.

```
[텍스트 LLM]
입력: 텍스트 → 출력: 텍스트
모델: Llama, GPT 등

[이미지 생성 AI]
입력: 텍스트 프롬프트 → 출력: 이미지
모델: Stable Diffusion, Flux, DALL-E 등
원리: "노이즈(잡음)에서 이미지를 점진적으로 복원" (Diffusion)
```

### 도구 선택: ComfyUI

이미지 생성 AI를 실행하는 인터페이스(UI)는 여러 가지가 있습니다:

| 도구 | 특징 | 추천 여부 |
|------|------|----------|
| **ComfyUI** | 노드 기반 워크플로우. 자유도 최고. 확장성 우수 | **추천 (메인)** |
| Automatic1111 WebUI | 전통적인 웹 UI. 간단하지만 유연성 낮음 | 보조/참고용 |
| Fooocus | 가장 간단. Midjourney처럼 프롬프트만 넣으면 됨 | 빠른 테스트용 |

**ComfyUI를 메인으로 추천하는 이유**:
- 이미지→영상, ControlNet, LoRA 등 **모든 확장이 노드로 연결** 가능
- Phase 3(영상)으로 자연스럽게 확장됨
- 커뮤니티 워크플로우를 그대로 가져다 쓸 수 있음
- 처음엔 복잡해 보이지만, 한번 익히면 가장 강력

### ComfyUI란? (좀 더 상세히)

일반적인 웹 UI와 달리, ComfyUI는 **노드(블록)를 연결**해서 이미지 생성 파이프라인을 만듭니다.

```
[텍스트 프롬프트] → [모델 로더] → [샘플러] → [디코더] → [이미지 출력]
                         ↑
                   [ControlNet 포즈]  ← 이런 식으로 블록 추가
```

각 블록이 하나의 기능을 담당하고, 이것들을 조합해서 복잡한 파이프라인을 만들 수 있습니다.

### 이미지 생성 모델 라인업

| 모델 | 특징 | VRAM 필요 | 검열 상태 |
|------|------|----------|----------|
| **SDXL** (Stable Diffusion XL) | 가장 성숙한 생태계, LoRA/ControlNet 풍부 | ~8GB | safety checker OFF 가능 |
| **Flux** | 최신 모델, 텍스트 렌더링 우수, 높은 품질 | ~10GB (Q4) | 커뮤니티 uncensored 버전 존재 |
| **커뮤니티 체크포인트** | CivitAI 등에서 배포. 특정 스타일/용도 특화 | 모델마다 다름 | 대부분 제한 없음 |

### 인체 구조 강화 도구

| 도구 | 하는 일 |
|------|--------|
| **ControlNet - OpenPose** | 관절/포즈를 스켈레톤으로 지정 → 그 포즈대로 이미지 생성 |
| **ControlNet - Depth** | 깊이감 제어 → 구도와 원근감 조절 |
| **ControlNet - Canny** | 윤곽선 기반 제어 → 형태 유지하며 스타일 변환 |
| **IP-Adapter** | 참조 이미지의 특징을 가져와서 새 이미지에 반영 |
| **FaceID** | 특정 얼굴을 일관되게 유지 |

- [ ] SDXL 또는 Flux 체크포인트 다운로드

## 6. Phase 3 — 영상 처리 (이미지→영상, 영상 보정)

> 목표: 이미지를 동영상으로 변환하거나, 기존 영상을 AI로 보정
> 비용: $0 (로컬 가능) 또는 RunPod (무거운 작업)

### 영상 처리의 두 가지 방향

#### A. 이미지 → 동영상 변환 (Image-to-Video)

정지 이미지에 움직임을 부여해서 짧은 영상(2~10초)을 만드는 것.

| 도구/모델 | 특징 | 비고 |
|----------|------|------|
| **AnimateDiff** | ComfyUI와 통합. SDXL 이미지에 움직임 추가 | 12GB로 가능 |
| **Stable Video Diffusion** | Stability AI의 공식 영상 모델 | 12GB로 가능 (해상도 제한) |

#### B. 기존 영상 AI 보정

| 작업 | 도구 | 설명 |
|------|------|------|
| **업스케일** | Real-ESRGAN, Topaz 등 | 저해상도 영상을 고해상도로 |
| **프레임 보간** | RIFE, FILM | 프레임 수를 늘려 부드러운 영상으로 |
| **스타일 변환** | ComfyUI 워크플로우 | 영상의 각 프레임을 AI로 스타일 변환 |

### Phase 3 체크리스트

- [ ] AnimateDiff ComfyUI 노드 설치
- [ ] 이미지→영상 변환 기본 테스트
- [ ] 움직임 품질/자연스러움 평가
- [ ] 영상 업스케일 도구 설치 및 테스트
- [ ] 프레임 보간 테스트
- [ ] VRAM 한계 파악 → RunPod 필요 여부 판단

> **이 Phase의 기술 상세는 Phase 2를 경험한 뒤 보강합니다.**
> ComfyUI에 익숙해진 상태에서 영상 확장이 훨씬 수월합니다.

## 7. Phase 4 — 평가 및 선택적 강화

> 목표: Phase 1~3에서 파악한 부족한 점을 **필요한 것만** 강화

이 Phase는 Phase 1~3의 결과에 따라 내용이 결정됩니다.
지금 시점에서 예상되는 강화 후보:

| 부족한 점 (예상) | 강화 방법 | 비용 |
|----------------|----------|------|
| 한국어 응답 품질 낮음 | 한국어 특화 모델로 교체, 또는 한국어 데이터 파인튜닝 | $0~$10 |
| 이미지에서 인체 비율이 이상 | 인체 특화 LoRA 적용, ControlNet 세팅 조정 | $0 |
| 특정 페르소나/캐릭터 필요 | 시스템 프롬프트 엔지니어링, 또는 캐릭터 LoRA 학습 | $0~$5 |
| 8B 모델 지능 부족 | 13B 모델로 교체, 또는 70B를 RunPod에서 실행 | $0.5/hr |
| 이미지 스타일 불만족 | CivitAI에서 스타일 LoRA 다운로드 또는 직접 학습 | $0~$5 |

**핵심 원칙**: 강화 방향은 **실제 사용 경험**에서 도출. 미리 추측해서 투자하지 않음.

그리고 이 시점에서 Phase 1에서 구동한 **uncensored LLM에게 직접 상담**하는 것이 효과적.
검열 없는 AI가 "이런 목적이면 이런 모델/데이터가 좋다"고 솔직하게 답해줄 수 있음.

## 9. 기술적 주의사항

### 검열 제거 관련

- **모델 레벨**: Abliterated/Dolphin 모델은 거절 메커니즘이 이미 제거됨
- **도구 레벨**: Ollama, ComfyUI 등 오픈소스 도구는 자체 검열이 없음
- **시스템 프롬프트**: 모델의 행동 양식은 시스템 프롬프트로 추가 제어 가능

### VRAM 관리 (12GB 기준)

- 텍스트 LLM과 이미지 생성을 **동시에 실행하면 VRAM 부족** 가능
- 한 번에 하나의 작업만 실행하거나, Ollama는 CPU 모드로 전환 가능
- `nvidia-smi` 명령어로 현재 VRAM 사용량 확인

### 모델/파일 관리

- 모델 파일은 수 GB~수십 GB 단위. 디스크 공간 확보 필요
- Ollama 모델 저장 경로: `C:\Users\[사용자]\.ollama\models\`
- ComfyUI 모델 저장 경로: `ComfyUI\models\checkpoints\` 등
- 안 쓰는 모델은 삭제해서 디스크 관리

## 11. 부록: ML 학습 프로젝트 로드맵

> 이 프로젝트(PrivateLLM)와는 별개로, ML/AI 기초를 체계적으로 배우기 위한 독립 프로젝트 제안.
> "밑바닥부터 파인튜닝"은 이 프로젝트가 아니라 여기서 다룸.

### 왜 별도 프로젝트인가?

PrivateLLM의 목적은 **"쓸 수 있는 AI를 빠르게 확보하는 것"** 이고,
ML 학습의 목적은 **"AI가 어떻게 작동하는지 이해하는 것"** 입니다.
목적이 다르므로 분리해야 각각에 집중할 수 있습니다.

### 추천 학습 경로 (단계별)

#### Step 1: 이론 기초 (1~2주)

| 리소스 | 내용 | 특징 |
|--------|------|------|
| **Andrej Karpathy - "Neural Networks: Zero to Hero"** (YouTube) | 신경망 기초부터 GPT까지 직접 코딩 | 전 OpenAI 연구원, 실습 중심, 무료 |
| **3Blue1Brown - "Neural Networks" 시리즈** (YouTube) | 직관적 시각화로 수학적 원리 설명 | 수학이 약해도 이해 가능, 무료 |

이 두 개만 보면 "LLM이 왜 이렇게 작동하는지" 대략 감이 옵니다.

#### Step 2: 실습 — 작은 모델 직접 학습 (1~2주)

| 프로젝트 | 내용 |
|---------|------|
| **텍스트 분류기 만들기** | 영화 리뷰 감정 분석 (긍정/부정). HuggingFace Transformers 입문 |
| **작은 LLM 파인튜닝** | GPT-2 (124M 파라미터)를 특정 스타일로 파인튜닝. 전체 과정 체험 |

이 단계에서 데이터 전처리, 학습 루프, loss, 평가 같은 개념을 실제로 다루게 됩니다.

#### Step 3: 본격 LLM 파인튜닝 (2~4주)

| 프로젝트 | 내용 |
|---------|------|
| **QLoRA로 8B 모델 파인튜닝** | Unsloth + 커스텀 데이터셋으로 Llama 8B를 직접 학습 |
| **LoRA로 이미지 모델 학습** | 특정 스타일/캐릭터를 학습시킨 LoRA 직접 제작 |

이 시점이 되면 PrivateLLM 프로젝트의 Phase 4(선택적 강화)에서 직접 파인튜닝을 할 역량이 생김.

#### Step 4: 심화 (선택)

- DPO (Direct Preference Optimization)
- RLHF (Reinforcement Learning from Human Feedback)
- 모델 아키텍처 이해 (Attention, Transformer)
- Kaggle 대회 참가

### 추천 도구/플랫폼

| 플랫폼 | 용도 | 비용 |
|--------|------|------|
| **Google Colab** | 무료 GPU로 실습 (T4 GPU) | 무료 (제한적) |
| **Kaggle Notebooks** | 무료 GPU + 데이터셋 접근 | 무료 (주 30시간) |
| **HuggingFace** | 모델/데이터셋 허브, Transformers 라이브러리 | 무료 |
| **RunPod** | 더 큰 GPU가 필요할 때 | $0.5/hr~ |
